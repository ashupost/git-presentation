git merge

Command: git merge <branch-name>

2 Types of merge:

Fast forward Three-way-merge
  • Simply moves the pointer forward
  • When there is no divergent work to merge
  • Conflict not possibale
  
  • If commits are merged which are not direct predecessors of the current branch
  • May be Conflict

Fast Forward merge

$ git checkout master (move HEAD to master)

$ git merge branch1 (master <---- branch1)

Fast Forward merge result

$ git checkout master (move HEAD to master)

$ git merge branch1 (master <---- branch1)

Three-way-merge

$ git checkout master (move HEAD to master)

$ git merge branch (master <---- branch)

Three-way-merge result

$ git checkout master (move HEAD to master)

$ git merge branch (master <---- branch)

Combines the respective changes from the two branches being merged. This commit points to both of its predecessors.

Also called the recursive merge strategy and is the default merge strategy.
